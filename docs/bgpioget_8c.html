Detailed Description

bgpioget. Executable to get gpio line values. This is just like gpioget but uses libbgpio instead of libgpio. The big difference between the 2 libraries is that libbgpio uses V2 system calls, where libgpio (currently) uses the deprecated V1 calls.

uint64_t  flags 
)
static

Open a gpio request. This specifies the base set of gpio line flags, as well as providing a string that can identify us as the holder of whatever lines we subsequently request.

Parameters
deviceA, possibly abbreviated, device spec. A full device spec would be something like "/dev/gpiochip0". This parameter may specify any final substring of this that uniquely matches a device, such as "gpiochip0", "chip0", or even just "0".
consumerThe name that will be associated with any gpio lines we reserve. This allows other users to figure out who is holding gpio lines.
flagsThe set of gpio line flags that should be applied by default to each gpio line we use. These can be individually set to other values for specific lines.
Returns
Pointer to dynamically allocated bgpio_request_t struct.
