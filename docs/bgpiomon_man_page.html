Man page for bgpiomon

NAME

bgpiomon - monitor gpio line values

SYNOPSIS

bgpiomon [OPTIONS] [gpiochip id] [line-spec]...

DESCRIPTION

Monitor GPIO lines for changes to input values.Options:

-b, --bias=[as-is|disable|pull-down|pull-up]

set the line bias (default=as-is)

-e, --edge=[rising|falling]:

set edge detection (default=rising)

-h, --help:

display this help message.

-l, --active-low, --low: make the line active-low.

-n, --name=name:

name for line reservation

-q, --quiet:

execute quietly

-r, --repeat=count

how many edges to detect (default=1)

-t, --timeout=millisecs

Specify an inactivity timeout period.

-v, --version:

display the version.

-x, --exec=path:

command to execute on detection

Gpiochip-ids may be a full path to the gpiochip device, or an abbeviated suffix (eg "chip0") of a valid path.

Line-specs are of the form N["["line-flag[,line-flag...]"]"]=B where line-flag may be a bias value, active-high, high or active-low, or an edge-detection value (rising, falling). N is the gpio line number and B is the binary digit 1 or 0,

eg "84[pull-up,high,rising]=1"

The command executed by the exec option will be passed the gpio device path, the gpio line number, the presumed new line value (1 for rising, 0 for falling), the event timestamp, the line sequence number, and the event sequence number.

The result of the command will be the value of the last event (1 or 0 as for exec), or an errorcode if an error occurred.
