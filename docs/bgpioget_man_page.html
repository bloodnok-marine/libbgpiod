Man page for bgpioget

NAME

bgpioget - get gpio input

SYNOPSIS

bgpioget [OPTIONS] <chip-id> <line-spec>...

DESCRIPTION

Get input from GPIO lines.

OPTIONS

-b, --bias=[as-is|disable|pull-down|pull-up]

set the line bias (default=as-is)

-d, --delta:

report only when state changes

-h, --help:

display this help message.

-l, --active-low, --low:

make the line active-low (default).

-n, --name=our_name:

who has reserved our gpio lines

-p, --period=usecs:

period for loop (default=2000000)

-q, --quiet:

execute quietly

-r, --repeat=count:

how many times to fetch (default=1)

-v, --version:

display the version.

-x, --exec=path:

command to execute on change

Gpiochip-ids may be a full path to the gpiochip device, or an abbeviated suffix (eg "chip0") of a valid path.

Line-specs are of the form N["["line-flag[,line-flag]"]"] where line-flag may be a bias value, active-high, high or active-low, eg 42[pull-down] 43[pull-up,active-high].

Specifying a repeat value of zero means repeat forever.

The command executed by the exec option will be passed the gpio device path, the gpio line number and the gpio line value as parameters.

The result of the command will be the value of the last successful gpio fetch, or an errorcode if an error occurred.
