Monitoring (Watching) GPIO Line States

To monitor a gpio line's state: whether it is reserved, whether it is configured for input or output, etc, you will need to open the chip using

  • bgpio_open_chip();
  • define which lines you wish to watch using bgpio_watch_line();
  • wait for their states to change using bgpio_await_watched_lines().

The command line tool bgpiowatch (source file bgpiowatch.c) provides a full working example of this.

A simpler example can be found in watch.c in the examples directory.

This is the main function from that file:

for (;;) {

Next, we figure out and report on what has changed:

switch (chg->event_type) {
case GPIO_V2_LINE_CHANGED_REQUESTED:
event = "requested";
break;
case GPIO_V2_LINE_CHANGED_RELEASED:
event = "released";
break;
case GPIO_V2_LINE_CHANGED_CONFIG:
event = "config changed";
break;
default:
fprintf(stderr,
"invalid event type received from the kernel\n");
bgpio_close_chip(chip);
return EXIT_FAILURE;
}
printf("line %u: %s at %" PRIu64 "\n",
chg->info.offset, event, (uint64_t)chg->timestamp_ns);
}

And finally, we close the device.
